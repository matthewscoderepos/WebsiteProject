TripTracker Android Application

1 / 9
Main app screen with menu shown. Four options shown on the left, weather information shown in the top right. Custom waypoint shown in middle screen.
2 / 9
Dialog shown when the user adds a custom waypoint. If the public option is selected the waypoint will be added to our database and shown to all users.
3 / 9
Dialog shown when the user starts a trip. The trip is named to allow the user to look at the trip details at a later point.
4 / 9
The waypoint manager which shows all custom waypoints and public waypoints in order by distance from the user. Local waypoints can be edited or deleted.
5 / 9
Option to add a waypoint is given in the manager as well. Users have the ability to use the current location and make the waypoint public.
6 / 9
The trip manager which shows the trips a user has taken in chronological order.
7 / 9
A details screen showing the selected trips: start and end times, average speed, and total distance. The user can show the trip on the map or delete the trip.
8 / 9
Showing a trip on the map. A re-center button is also shown which appears when the user moves the camera.
9 / 9
The very barebones weather screen is shown here. Current weather and conditions are shown. Uses the current location of the user to pull weather data if the data has not been pulled recently.

An Android application written in Java. Provides the user with full control of a Google Map, private and public waypoint creation, trip tracking including the start and endtimes, speed and distance of the trip, and shows the current local weather conditions.
